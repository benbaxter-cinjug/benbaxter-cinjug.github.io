Clean:www


gulp.task('clean:www', function () {
  del(['www/**', '!www']);
})


Notice the globbing

This keeps the www directory
but deletes all of the contents inside

The colon (:) is gulp convention for defining sub tasks...

Dependencies


	gulp.task('clean', ['clean:www', 'clean:dist', 'clean:upForDinner']);

clean:www & clean:dist & clean:upForDinner
will all be run before clean

var rename = require('gulp-rename');
var uglify = require('gulp-uglify');

gulp.task('minify:js', function() {
	gulp.src('src/js/**/*.js')
	    // This will output the non-minified version
	    .pipe(gulp.dest("stage/js/"))
	    // This will minify and rename to *.min.js
	    .pipe(uglify())
	    .pipe(rename({ extname: '.min.js' }))
	    .pipe(gulp.dest("stage/js/"));
})

Lessons Learned

  • async is tough.
    • Order can matter during a build
  • Plugins for everything
  • Too many plugins...
  • Often two or three plugins to replace 'old' ones
  • gulp-clean vs. gulp-rimraf vs. del
  • gulp-merge vs. gulp-sequence
  • Many uses, can do about anything

Helpful Plugins

  • css/sass/less plugins like gulp-minify-css
  • gulp-shell: Run shell command
  • Mobile
    • cordova: Run cordova commands programatically
    • plist: ios plist generation
  • gulp-zip: Zip up code
  • gulp-load-plugins a.k.a plugin abstraction
  • jshint plugin
  • karma and testing plugins
